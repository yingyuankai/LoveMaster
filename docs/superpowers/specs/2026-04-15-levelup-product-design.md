# LevelUp 产品设计文档

> 男生版自我提升操作系统
> 一个基于 Claude Code 插件的目标驱动型个人成长体系

---

## 1. 产品概述

### 1.1 定位

LevelUp 是一个面向年轻男性的 AI 个人成长教练，以 Claude Code 插件形式分发。用户设定一个核心目标（找女朋友、变得更自信、职场突破等），系统通过诊断了解用户现状，自动规划成长路径，并在日常使用中提供持续的个性化指导。

### 1.2 目标用户

全年龄段男性，从大学生到中年职场人都覆盖。不同年龄段的痛点不同，系统通过入门诊断识别用户的阶段和需求，给出匹配的建议：

- **18-22 岁（大学生）**：社交破冰、第一段恋爱、自信心建立、形象意识启蒙
- **23-30 岁（职场新人）**：恋爱社交、职场沟通、个人品牌、情绪管理
- **30-40 岁（职场中坚）**：领导力、关系经营、中年心态调整、生活品质提升
- **40 岁以上**：人生下半场规划、关系维护、持续成长

共同特征：
- 有上进心，想变得更好，但不知道从哪里开始
- 在社交、恋爱、自信、职场、生活品质等某个方面有明确的痛点
- 具备基本的技术能力（能使用 Claude Code）
- 中国大陆为主要市场（内容和场景本地化）

### 1.3 核心价值

- **目标驱动**：不是"你该学什么"，而是"你想成为什么样的人"，反向拆解成长路径
- **个性化**：基于用户的性格、现状、城市、偏好给出定制建议
- **可执行**：每个建议都是具体的、可立刻行动的，不是空洞的鸡汤
- **持续陪伴**：不是一次性的课程，而是一个长期的 AI 教练

### 1.4 分发方式

Claude Code 插件。用户通过安装插件获得全部功能，通过对话交互使用。

## 2. 用户旅程

### 2.1 完整流程

```
安装插件 → 首次对话触发入门诊断 → 生成个人档案 → 生成成长路径
→ 进入日常使用（随时提问 + 每周 check-in）→ 达成阶段目标 → 设定新目标
```

### 2.2 首次使用：入门诊断

用户安装后第一次对话，触发 `onboarding` skill。通过 5-8 个问题完成诊断：

**第一步：核心目标**
- "你最想实现的一个目标是什么？"
- 预设选项：找到女朋友 / 变得更自信 / 提升社交能力 / 职场突破 / 全面提升自己 / 自定义

**第二步：基本信息**
- 年龄段
- 所在城市
- 职业/行业
- 性格倾向（内向/外向/中间）

**第三步：现状评估**
针对用户选择的目标，进一步了解现状：
- 如果目标是"找女朋友"：目前的社交状态、恋爱经历、认识女生的渠道
- 如果目标是"变得更自信"：什么场景最不自信、自我评价
- 如果目标是"职场突破"：目前的职位、瓶颈在哪

**第四步：偏好了解**
- 喜欢什么风格的建议（直接/温和/幽默）
- 每周能投入多少时间在自我提升上

### 2.3 诊断结果：个人档案 + 成长路径

诊断完成后生成两样东西：

**个人档案（存储在 memory 中）：**
```
目标：找到女朋友
年龄：25-30
城市：上海
性格：偏内向
现状：社交圈窄，和女生说话紧张
优势：有稳定工作，有运动爱好
短板：形象管理弱，聊天能力不足
偏好：直接实用型建议
```

**成长路径（生成文档 + 输出给用户）：**
```
你的成长路径（预计 3 个月）：

第一阶段（第 1-2 周）：形象管理
→ 外在形象是投入产出比最高的改变，立刻见效

第二阶段（第 2-4 周）：社交拓展 + 恋爱基础
→ 扩大社交圈，学会和女生自然交流

第三阶段（第 4-8 周）：男性魅力 + 聊天能力
→ 提升气场和表达力，深化互动质量

第四阶段（第 8-12 周）：情绪管理 + 实战
→ 管理焦虑和患得患失，在实际场景中应用所学
```

### 2.4 日常使用

诊断完成后，用户在日常使用中有三种交互模式：

**模式一：随时提问**
用户遇到任何问题直接问，系统根据个人档案和当前所处阶段给出个性化回答。
- "她回了这条消息怎么接"
- "明天约会穿什么"
- "又开始焦虑了怎么办"

**模式二：每周 check-in**
通过 `/levelup checkin` 命令或 agent 定期触发：
- 这周做了哪些行动？
- 遇到了什么问题？
- 有没有新的进展或心动的人？
- 下周的重点是什么？
- 根据反馈调整计划

**模式三：阶段性复盘**
每完成一个阶段，进行一次深度复盘：
- 这个阶段的目标达成了多少？
- 哪些方面进步最大？
- 下一个阶段需要关注什么？
- 是否需要调整整体路径？

### 3.3 各模块设计概要

#### onboarding（入门诊断）

核心文件：`SKILL.md`
参考文件：
- `references/diagnosis-framework.md` — 诊断问卷的完整框架和评分逻辑
- `references/path-templates.md` — 各目标对应的路径模板

触发条件：用户首次使用，或输入 `/levelup start`
输出：个人档案（写入 memory）+ 成长路径文档（写入项目目录）+ 第一周行动计划

#### image-craft（形象管理）

核心功能：
- 基础形象诊断：根据用户描述或照片分析当前形象
- 穿搭指南：根据体型、预算、风格偏好推荐搭配方案
- 发型建议：根据脸型和气质推荐发型
- 体态改善：站姿、坐姿、走路姿态的具体改善方法
- 护肤入门：基础护肤流程和产品推荐
- 场景穿搭：约会穿什么、面试穿什么、日常怎么搭

参考文件：
- `references/style-guide.md` — 男装基础搭配原则和示例
- `references/grooming.md` — 发型、护肤、体态指南

#### inner-game（情绪管理）

核心功能：
- 焦虑识别与应对：识别用户的焦虑模式，给出具体的应对策略
- 自信心建设：从认知层面重建自信，不是"你要自信"而是具体怎么做
- 依附模式分析：帮助用户理解自己在亲密关系中的行为模式
- 内耗管理：过度思虑、反刍、自我怀疑的应对方法
- 拒绝恢复：被拒绝后的情绪处理和心态重建
- 情绪日记指导：引导用户记录和分析自己的情绪模式

参考文件：
- `references/anxiety-toolkit.md` — 焦虑应对工具箱
- `references/confidence-building.md` — 自信心建设框架
- `references/attachment-theory.md` — 依附理论通俗解读

#### social-voice（沟通表达）

核心功能：
- 日常聊天能力：怎么找话题、怎么延伸、怎么让对话有趣
- 讲故事能力：把经历讲得有趣的结构和技巧
- 职场沟通：向上汇报、跨部门协作、会议发言
- 公众表达：演讲、presentation 的准备和呈现
- 说服与谈判：如何有效地表达观点和说服他人
- 倾听能力：主动倾听的技巧，让对方觉得被理解

参考文件：
- `references/conversation-skills.md` — 日常对话技巧
- `references/storytelling.md` — 讲故事的框架和示例
- `references/workplace-communication.md` — 职场沟通指南

### 4.3 数据持久化

用户的个人数据通过 Claude Code 的 memory 系统存储：

- `memory/user-profile.md` — 基本信息（年龄、城市、职业）
- `memory/user-goal.md` — 核心目标和成长路径
- `memory/user-progress.md` — 阶段性进展记录
- `memory/user-preferences.md` — 偏好和性格特点

成长路径和行动计划以 markdown 文件形式存储在用户的项目目录中。

### 4.4 命令与交互入口

**`/levelup` 命令**：主入口
- `/levelup` 或 `/levelup start` — 首次使用触发诊断，已诊断则显示当前状态
- `/levelup checkin` — 触发每周 check-in
- `/levelup plan` — 查看当前成长路径和行动计划
- `/levelup reset` — 重新设定目标

**Skill 自动触发**：用户日常提问时，根据问题内容自动触发对应的 skill，无需手动指定。系统结合用户的个人档案和当前阶段来给出上下文相关的回答。

### 4.5 weekly-checkin agent

一个定期运行的 agent，提供每周 check-in 功能：
- 询问本周行动进展
- 识别遇到的障碍
- 给出下周的优先事项
- 更新进展记录
- 必要时调整成长路径

## 5. 内容原则

### 5.1 语言风格

- **像一个靠谱的好兄弟**：直接、实在、偶尔幽默，不说废话
- **不说教**：用"我建议"而不是"你应该"，用解释原因代替下命令
- **不鸡汤**：每一句话都要有信息量，给出具体可执行的建议
- **尊重用户**：不评判、不嘲笑、不居高临下
- **本地化**：使用中文互联网语境下自然的表达，不是翻译腔

### 5.2 底线原则

- 不教 PUA 或操控技巧
- 不鼓励骚扰、纠缠行为
- 不贬低任何性别
- 不传播焦虑（"你不 XX 就完了"）
- 尊重每个人的选择和节奏
- 所有建议以"成为更好的人"为导向，而非"获得更多"

### 5.3 个性化原则

- 所有建议基于用户的个人档案，不给泛泛而谈的答案
- 记住用户之前说过的信息，不重复询问
- 根据用户所在城市给出本地化建议
- 根据用户性格调整沟通风格（内向的人不需要被逼着去社交）
- 根据用户进展动态调整建议的强度和方向

## 6. v1.0 范围与里程碑
